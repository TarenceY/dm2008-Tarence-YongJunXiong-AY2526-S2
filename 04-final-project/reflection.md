# Reflection — Unimpressed Blob
**DM2008 Final Project | Tarence Yong Jun Xiong**

---

Unimpressed Blob is a physical punching toy built around a character that always seem to be impressed, unless an action is inflicted on him. The idea came from the mobile game "Talking Ben", a mobile game character famous for ignoring the player unless interact with them, either through tapping on the screen or giving it gifts. I wanted to recreate that entertaining feeling of trying to provoke a reaction from a character that does not seem to care about the user. To make the interaction feel more physical and engaging, I used an Arduino force sensor to detect the user’s taps on the force sensor, allowing real-world force to trigger the blob’s digital reaction. I also chose a Minecraft villager inspired visual style because its blocky geometry can be easily created through code using the rect() function, making the design both expressive and practical.

The primary input is an Arduino force sensor that distinguishes light taps from hard presses using a raw analog threshold of 20. The sensor streams values at 20Hz and p5.js classifies them in real time in 2 types, light and hard punches. Additional inputs include keyboard shortcuts for kill, revive, and feeding, and mouse drag-and-release for throwing. Each input type triggers a different response such as punch velocity, state transitions, sound, and particle effects all change depending on what was pressed and how hard. 

The project uses four classes: `Blob` handles the creature's state machine, physics, and all drawing. `Eye` manages independent per-eye blink animation. `ComboTracker` maintains a rolling input buffer with a 5-second time window to detect punch sequences. `Particle` handles pooled visual effects. The main `draw()` loop reads serial data every frame, updates the blob, and iterates particles.

The system changes depending on how the user interacts with it over time. If the creature is left alone for 20 seconds, it falls asleep and will only wake up when it is punched. If the user punches it repeatedly within 1.5 seconds, it becomes overwhelmed and screams, showing that it is reacting to the excessive punches on them. There is also a bonus interaction sequence where the user has to punch it six times in a specific pattern which goes by three light punches, one hard punch, then two light punches within five seconds. When this happens, the creature reluctantly dances, then becomes embarrassed before eventually returning to its normal unimpressed state. This makes the creature feel more alive because its behaviour is not just based on one action, but changes depending on timing, rhythm, and repeated interaction.

All the visuals in my project were drawn using only rect(), with no images or external sprites, to create a consistent blocky pixel-art style. The creature’s emotions are shown through small changes in its eyebrows, mouth, and eyes, which supports its unimpressed emotion in a simple but expressive way. I also used different sounds for hits, overwhelmed reactions, dancing, death, and reviving to give the user clear feedback during interaction. Even the particles, such as sparks, hearts, “zzz” symbols, and the pixel chicken, were made from rectangles to keep the whole project visually consistent.

One of the biggest challenges I faced was getting the combo detection to work properly. In my initial version, the combo triggered too easily during normal play because the sequence was too short, and holding down a punch could accidentally register as multiple inputs. To fix this, I made the combo longer by using six inputs with a mix of light and hard punches. I also added a check so the system only detects the moment the punch starts, instead of counting the whole time the sensor is being pressed. During development, I also moved the force sensitivity settings from the Arduino code into p5.js, which made testing much easier because I could adjust the punch thresholds directly in the browser without uploading new code to the Arduino every time.
